Getting started

Editron is a Claude Code skill that edits video by conversation. You drop footage in a folder, start a Claude Code session, and describe what you want. Editron transcribes your takes, proposes a cut strategy, and produces a polished edit once you confirm.

Install

Editron runs as a skill inside Claude Code. Clone the repo, symlink it into your skills directory, install the Python dependencies, and set your ElevenLabs API key.

git clone https://github.com/mathisdittrich/editron.git
cd editron/video-use-main
ln -s "$(pwd)" ~/.claude/skills/video-editor
pip install -e .
cp .env.example .env      # paste ELEVENLABS_API_KEY=...

You'll also need ffmpeg on your PATH. Optional: yt-dlp for downloading online sources, and Manim or Remotion if you want programmatic animation overlays.

Your first edit

Put your footage in a folder — anywhere. Start a Claude Code session from that folder and tell Editron what to make.

cd ~/footage/launch-video
claude

> edit these into a 60s launch video

Editron will probe every file, transcribe in parallel, and write a phrase-level packed transcript. Then it will ask questions shaped by what it sees — content type, target length, aesthetic, must-preserve moments — and propose a strategy in four to eight sentences.

Confirm in plain English. Editron produces a preview at edit/preview.mp4. Iterate with natural-language feedback. Final render on the word ship.

Environment

The only required environment variable is ELEVENLABS_API_KEY. Editron loads it from .env at the project root or from your shell environment.

ELEVENLABS_API_KEY=sk_...

The rest is automatic — Editron caches transcripts per source file, so you never re-transcribe unless the source changed.

The loop

Every session follows the same shape: inventory → converse → strategy → execute → iterate → persist. The steps are not a menu, they are a loop. Editron never touches the cut before you've approved the plain-English plan.

Strategy step

Before any render, Editron writes a short strategy note describing the intended shape (hook → problem → solution → CTA, for example), the takes it plans to use, the cut direction, the grade, any animations, and the subtitle style. Length estimate in seconds.

You confirm or redirect. No edits happen until confirmation.

Iteration

After the first preview, iterate with natural-language feedback:

  • "The hook is too slow. Tighten it."
  • "Use take 4 instead of take 2 for the punchline."
  • "Warmer grade. Push the shadows."
  • "Lose the subtitle on the final shot — let it breathe."

Editron re-plans, re-renders, and shows you the new preview. Transcripts are cached, so iteration is fast — only the render pipeline runs.

Hard rules

These are the twelve production-correctness rules Editron encodes. They are not taste — they are the difference between a cut that works and one with silent failures.

  1. Subtitles are applied last in the filter chain.
  2. Per-segment extract → lossless -c copy concat. No single-pass filtergraph.
  3. 30ms audio fades at every segment boundary.
  4. Overlays use setpts=PTS-STARTPTS+T/TB to shift frame 0 to the window start.
  5. Master SRT uses output-timeline offsets.
  6. Never cut inside a word.
  7. Pad every cut edge (30–200ms working window).
  8. Word-level verbatim ASR only.
  9. Cache transcripts per source.
  10. Parallel sub-agents for multiple animations.
  11. Strategy confirmation before execution.
  12. All session outputs in <videos_dir>/edit/.

EDL format

The edit decision list is a JSON document with sources, ranges, grade, overlays, and subtitles:

{
  "version": 1,
  "sources": {"C0103": "/abs/path/C0103.MP4"},
  "ranges": [
    {"source": "C0103", "start": 2.42, "end": 6.85,
     "beat": "HOOK", "quote": "...", "reason": "..."}
  ],
  "grade": "warm_cinematic",
  "overlays": [...],
  "subtitles": "edit/master.srt",
  "total_duration_s": 87.4
}

Session memory

Every session appends a section to <videos_dir>/edit/project.md. Strategy, decisions, reasoning log, outstanding items. Next week's session reads it first and summarizes the last run in one sentence before asking whether to continue.
